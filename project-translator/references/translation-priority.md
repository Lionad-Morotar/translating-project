# 翻译优先级规则

## 概述

按照以下优先级顺序选择文件进行翻译，确保先翻译最重要的项目文档和核心代码。

## 优先级顺序

### 1. 项目入口文档
- `README.md`
- `CHANGELOG.md`

### 2. 项目资料
- `docs/` 目录下的文档
- 项目说明文档

### 3. 核心代码
- `src/` 目录
- `lib/` 目录
- `app/` 目录
- 其他核心源码目录

### 4. 其他文件
- 按文件路径顺序翻译其他文件

## 禁止翻译

- 禁止翻译 node_modules 文件夹

## 翻译策略

- **入口文档优先**：让用户快速了解项目整体
- **资料文档次之**：理解项目设计和使用方法
- **核心代码再次**：理解项目实现细节
- **其他文件最后**：补充性文档和配置文件

## 实现方式

智能体在读取任务清单后，应按照上述优先级重新排序待翻译文件，然后逐个处理。
