### 测试结构

1. **测试套件**: `e2e` 端到端测试

2. **环境设置** (beforeEach):

   * 创建两个 GitHub 仓库：`test-source-repo` 和 `test-target-repo`

   * Source 仓库包含测试文件：

     * README.md（英文文档）

     * src/utils.js（JavaScript 代码文件）

     * docs/api.md（API 文档）

   * Target 仓库初始化为翻译项目

3. **环境清理** (afterEach):

   * 删除临时创建的 GitHub 仓库

   * 清理本地临时目录

4. **测试用例**（仅描述，暂不实现功能）:

   * 测试扫描项目生成任务清单

   * 测试更新任务状态

### 技术实现

* 使用 `mcp_GitHub_create_repository` 创建仓库

* 使用 `mcp_GitHub_push_files` 提交初始文件

* 使用 `mcp_GitHub_delete_file` 清理文件（或删除整个仓库）

* 使用 ESM 模块语法（与现有测试保持一致）

* 使用中文注释和描述
